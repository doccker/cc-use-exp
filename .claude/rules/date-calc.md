# 日期计算规则

> 防止 LLM 在日期加减时默认使用"月末对齐"逻辑。

---

## 默认行为

"N 个月"、"N 个月后"、"账期 N 个月" = **日期 + N 月，保持日不变**

| 输入 | 正确结果 | 错误结果 |
|------|---------|---------|
| 2025-09-27 + 3 个月 | 2025-12-27 | ~~2025-12-31~~（月末） |
| 2025-10-24 + 3 个月 | 2026-01-24 | ~~2026-01-31~~（月末） |
| 2026-03-15 - 1 个月 | 2026-02-15 | ~~2026-02-28~~（月末） |

## 月末对齐仅在明确要求时使用

仅当需求中出现以下关键词时，才使用月末对齐：
- "月末"、"月底"、"自然月月末"、"endOfMonth"、"到月底"

## 实现要求

使用语言原生日期 API，禁止手动计算月份天数：
- Java: `LocalDate.plusMonths(N)`
- Go: `time.AddDate(0, N, 0)`
- JS/TS: `dayjs().add(N, 'month')` 或 `date-fns/addMonths`
